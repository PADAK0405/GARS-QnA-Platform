############################################
# .env 파일 생성 가이드
############################################

프로젝트 루트에 .env 파일을 생성하고 아래 내용을 복사하세요:

-------- .env 파일 내용 시작 --------

# Server Configuration
PORT=3000
NODE_ENV=development

# Google OAuth (https://console.cloud.google.com/apis/credentials 에서 발급)
GOOGLE_CLIENT_ID=여기에_발급받은_클라이언트_ID를_입력하세요
GOOGLE_CLIENT_SECRET=여기에_발급받은_클라이언트_보안_비밀을_입력하세요
CALLBACK_URL=http://localhost:3000/auth/google/callback

# Session Secret (랜덤 문자열 - 프로덕션에서는 반드시 변경)
# 보안을 위해 최소 32자 이상의 복잡한 문자열 사용 권장
SESSION_SECRET=super_secret_session_key_change_this_in_production_minimum_32_chars

# 보안 설정
# Rate Limiting 설정 (선택사항)
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX_REQUESTS=100

# CORS 설정 (프로덕션에서 도메인 제한)
ALLOWED_ORIGINS=http://localhost:3000,https://yourdomain.com

# Google Gemini AI (https://aistudio.google.com/app/apikey 에서 발급)
GEMINI_API_KEY=여기에_발급받은_API_키를_입력하세요

# MySQL Database Configuration (로컬 개발환경)
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_NAME=qna_hub

# Connection Pool Settings
DB_CONNECTION_LIMIT=10
DB_QUEUE_LIMIT=0

-------- .env 파일 내용 끝 --------

############################################
# AWS 프로덕션 환경 설정
############################################

AWS RDS 사용 시 아래 값들을 변경하세요:

NODE_ENV=production
DB_HOST=your-rds-endpoint.rds.amazonaws.com
DB_USER=admin
DB_PASSWORD=your_secure_rds_password
CALLBACK_URL=https://yourdomain.com/auth/google/callback

############################################
# 보안 주의사항
############################################

1. .env 파일은 절대 Git에 커밋하지 마세요!
2. SESSION_SECRET은 복잡한 랜덤 문자열로 변경하세요 (최소 32자)
3. 프로덕션 환경에서는 강력한 DB 비밀번호 사용
4. AWS IAM 역할을 사용하여 자격증명 관리 권장
5. HTTPS 사용 (프로덕션 환경 필수)
6. 정기적인 보안 업데이트 및 취약점 점검
7. Rate Limiting 설정으로 DDoS 공격 방지
8. 입력 데이터 검증 및 XSS/CSRF 방지
9. 파일 업로드 시 보안 검사 필수
10. 관리자 권한 최소 권한 원칙 적용
